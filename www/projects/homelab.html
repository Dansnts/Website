Stack technique

K3s Kubernetes léger avec Traefik intégré, Ubuntu 24.04
Proxmox Hyperviseur. VMs provisionnées via Terraform (bpg/proxmox ~0.73)
TrueNAS Scale ZFS + SMB, 3 pools (NAS SSD 1TB, Media HDD 6TB, Logs HDD 1TB)
Ansible Configuration idempotente : Proxmox, K3s node, node_exporter
cert-manager + Infomaniak Wildcard *.<domain> via Let's Encrypt DNS-01
MetalLB LoadBalancer bare-metal, pool 10.0.0.100–110
SMB CSI driver PVC Kubernetes montés sur les partages TrueNAS
ELK 8.x Elasticsearch StatefulSet + Logstash + Kibana + Filebeat DaemonSet
rclone CronJob Backup quotidien 2h, NAS → Google Drive

Déploiement

Un script installe tout dans l'ordre correct : CSI driver SMB, cert-manager + webhook Infomaniak, namespace, certificat wildcard (attend Ready), MetalLB, puis tous les services.

k8s/deploy.sh Bash 
#!/bin/bash
set -euo pipefail

# 1. CSI driver SMB
curl -skSL https://raw.githubusercontent.com/kubernetes-csi/csi-driver-smb/v1.16.0/deploy/install-driver.sh \
  | bash -s v1.16.0 --

# 2. cert-manager + webhook Infomaniak
kubectl apply -f domain-name/

# 3. Namespace + secrets
kubectl apply -f namespace.yaml
kubectl apply -f smb-secret.yaml

# 4. Certificat wildcard (attend Ready)
kubectl wait certificate fariadossantos-wildcard \
  -n homelab --for=condition=Ready --timeout=300s

# 5. MetalLB
kubectl apply -f metallb.yaml

# 6. Services
for svc in elk immich jellyfin backup teamspeak root pihole wireguard grafana prometheus; do
  kubectl apply -f $svc/
done

# 7. Ingress
kubectl apply -f ingress.yaml

kubectl get pods -n homelab

Réseau

DNS (Pi-hole v6)

Pi-hole tourne sur 10.0.0.101. Depuis le réseau maison (192.168.1.x), MikroTik forwarde le port 53 vers Pi-hole. Le node K3s utilise 1.1.1.1 directement (pas Pi-hole : chicken-and-egg au boot).

Certificats TLS

Wildcard *.<domain> via cert-manager (Let's Encrypt DNS-01, webhook Infomaniak). Renouvellement automatique tous les ~60 jours. TrueNAS et Proxmox utilisent le wildcard importé manuellement (~90 jours).

export cert → TrueNAS / Proxmox Bash 
# Exporter depuis K8s
kubectl get secret fariadossantos-wildcard-tls -n homelab \
  -o jsonpath='{.data.tls\.crt}' | base64 -d > tls.crt

kubectl get secret fariadossantos-wildcard-tls -n homelab \
  -o jsonpath='{.data.tls\.key}' | base64 -d > tls.key

# TrueNAS : Credentials → Certificates → Import Certificate
# Proxmox : Node → System → Certificates → Upload Custom Certificate

Backup NAS → Infomaniak Swiss Backup (Restic)

Restic est un outil de backup incrémental avec déduplication et chiffrement zero-knowledge côté client (AES-256). Le stockage S3 est hébergé en Suisse chez Infomaniak. Les blobs uploadés sont entièrement opaques, la clé ne quitte jamais le cluster.

Chiffrement AES-256 côté client avant envoi, Infomaniak ne voit que des blobs opaques
Rétention 7 jours / 4 semaines / 6 mois / 2 ans, restic forget --prune automatique
Source PVC backup-source monté en lecture seule depuis TrueNAS via SMB
Planification CronJob chaque nuit à 02h00 (Europe/Zurich), concurrencyPolicy: Forbid
k8s/backup/restic-cronjob.yaml YAML 
apiVersion: batch/v1
kind: CronJob
metadata:
  name: restic-backup
  namespace: homelab
spec:
  schedule: "0 2 * * *"
  concurrencyPolicy: Forbid
  jobTemplate:
    spec:
      backoffLimit: 1
      template:
        spec:
          restartPolicy: OnFailure
          containers:
            - name: restic
              image: restic/restic:latest
              command: ["/bin/sh", "-c"]
              args:
                - |
                  # Init repo S3 si inexistant
                  restic snapshots > /dev/null 2>&1 || restic init

                  restic backup /mnt/data/Backup \
                    --tag auto --ignore-inode \
                    --exclude '/mnt/data/Backup/Games/Pok*'
                  # --ignore-inode : évite les faux positifs sur SMB (inodes non stables)

                  restic forget \
                    --keep-daily 7 --keep-weekly 4 \
                    --keep-monthly 6 --keep-yearly 2 \
                    --prune --tag auto

                  restic check
              envFrom:
                - secretRef:
                    name: restic-credentials  # gitignored
              volumeMounts:
                - name: data
                  mountPath: /mnt/data
                  readOnly: true
          volumes:
            - name: data
              persistentVolumeClaim:
                claimName: backup-source
backup manuel Bash

GitOps avec ArgoCD

Tout changement infra passe par Git. ArgoCD synchronise automatiquement l'état du cluster sur l'état déclaré dans le repo, pas de kubectl apply manuel en production.

1. Résolution du conflit de sync sur StorageClass

ArgoCD utilisait la stratégie Apply par défaut. Les StorageClass ont des champs immuables : ArgoCD retournait une erreur de réconciliation en tentant de patcher des paramètres qu'il ne peut pas modifier.

Fix : annotation Replace=true. ArgoCD supprime et recrée la ressource au lieu de patcher, ce qui contourne la contrainte d'immutabilité de l'API Kubernetes.

k8s/storageclass-smb.yaml YAML 
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: smb-nas
  annotations:
    argocd.argoproj.io/sync-options: Replace=true  # delete + recreate (champs immuables)
provisioner: smb.csi.k8s.io
parameters:
  source: "//10.0.0.11/NAS"
  csi.storage.k8s.io/node-stage-secret-name: smb-secret
  csi.storage.k8s.io/node-stage-secret-namespace: homelab
reclaimPolicy: Retain
volumeBindingMode: Immediate

2. Exposition de l'UI ArgoCD via Ingress Traefik + TLS

ArgoCD server exposé en HTTPS sur argocd.<domain> via Traefik (entrypoint websecure, port 443). TLS assuré par le wildcard *.<domain> géré par cert-manager. DNS résolu localement par Pi-hole (enregistrement A → 10.0.0.100).

k8s/argocd/ingress.yaml YAML 
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: argocd-ingress
  namespace: argocd
  annotations:
    traefik.ingress.kubernetes.io/router.entrypoints: websecure
    traefik.ingress.kubernetes.io/router.tls: "true"
    traefik.ingress.kubernetes.io/router.middlewares: homelab-argocd-headers@kubernetescrd
spec:
  tls:
    - hosts: [argocd.<domain>]
      secretName: fariadossantos-wildcard-tls
  rules:
    - host: argocd.<domain>
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: argocd-server
                port: { number: 80 }
