Homelab Infrastructure

Infrastructure Kubernetes personnelle : media, monitoring, logs, backups

Vue d'ensemble

Cluster Kubernetes single-node (Ubuntu 192.168.1.16) avec stockage NAS FreeNAS via SMB CSI driver. Tous les services sont déclarés en YAML, versionnés et déployables en une commande. Les backups sont automatisés vers Google Drive via rclone CronJob.

Services hébergés

  • Jellyfin : Media server (500 Gi NAS en read-only)
  • ELK Stack : Elasticsearch + Logstash + Kibana + Filebeat DaemonSet
  • Prometheus + node-exporter : Métriques cluster et système
  • Grafana : Dashboards et visualisation
  • TeamSpeak : Serveur vocal (NodePort)
  • Backup rclone : CronJob quotidien NAS → Google Drive
  • ROOT : Application personnelle exposée avec TLS Let's Encrypt

Déploiement automatisé

Un seul script installe le CSI driver SMB, crée le namespace, et déploie l'ensemble des services dans l'ordre correct, en attendant qu'Elasticsearch soit healthy avant de démarrer Logstash.

deploy.sh Bash 
#!/bin/bash
# Déploiement complet du homelab Kubernetes
set -euo pipefail

echo "=== Installation du CSI driver SMB ==="
curl -skSL https://raw.githubusercontent.com/kubernetes-csi/csi-driver-smb/v1.16.0/deploy/install-driver.sh \
  | bash -s v1.16.0 --

echo "=== Création du namespace ==="
kubectl apply -f namespace.yaml

echo "=== Création du secret SMB ==="
read -p "Secret SMB configuré ? (y/n) " -n 1 -r; echo
if [[ $REPLY =~ ^[Yy]$ ]]; then
  kubectl apply -f smb-secret.yaml
fi

echo "=== Déploiement Jellyfin ==="
kubectl apply -f jellyfin/

echo "=== Déploiement Backup CronJob ==="
kubectl apply -f backup/configmap.yaml
kubectl apply -f backup/pvc.yaml
kubectl apply -f backup/cronjob.yaml

echo "=== Déploiement ELK Stack ==="
kubectl apply -f elk/elasticsearch.yaml
echo "Attente Elasticsearch ready..."
kubectl wait --for=condition=ready pod -l app=elasticsearch \
  -n homelab --timeout=300s
kubectl apply -f elk/logstash.yaml
kubectl apply -f elk/kibana.yaml
kubectl apply -f elk/filebeat.yaml

kubectl get pods -n homelab

Backup NAS → Google Drive

CronJob Kubernetes qui exécute rclone chaque nuit à 3h UTC. Le montage NAS est en read-only, le token Google Drive est stocké dans un Secret K8s. concurrencyPolicy: Forbid garantit qu'un seul job tourne à la fois.

backup/cronjob.yaml YAML 
apiVersion: batch/v1
kind: CronJob
metadata:
  name: nas-backup-gdrive
  namespace: homelab
spec:
  schedule: "0 3 * * *"         # 3h UTC quotidien
  concurrencyPolicy: Forbid    # pas de job simultané
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 3
  jobTemplate:
    spec:
      backoffLimit: 2
      template:
        spec:
          restartPolicy: OnFailure
          containers:
            - name: rclone
              image: rclone/rclone:latest
              command: ["/bin/sh", "/scripts/backup.sh"]
              volumeMounts:
                - name: script
                  mountPath: /scripts
                - name: rclone-config
                  mountPath: /config/rclone
                  readOnly: true
                - name: nas-data
                  mountPath: /mnt/nas
                  readOnly: true
              resources:
                requests: { cpu: 250m, memory: 256Mi }
                limits:   { cpu: "1",   memory: 512Mi }
          volumes:
            - name: script
              configMap:
                name: backup-script
                defaultMode: 0755
            - name: rclone-config
              secret:
                secretName: rclone-config   # token OAuth Google Drive
            - name: nas-data
              persistentVolumeClaim:
                claimName: backup-source    # 1Ti SMB FreeNAS

Collecte centralisée des logs : Filebeat DaemonSet

Filebeat tourne sur chaque nœud en tant que DaemonSet, collecte tous les logs des containers depuis /var/log/containers/, enrichit avec les métadonnées Kubernetes, et envoie à Logstash. Le ClusterRole donne accès en lecture à l'API K8s pour l'enrichissement.

elk/filebeat.yaml YAML 
apiVersion: v1
kind: ConfigMap
metadata:
  name: filebeat-config
  namespace: homelab
data:
  filebeat.yml: |
    filebeat.inputs:
      - type: container
        paths:
          - /var/log/containers/*.log
        processors:
          - add_kubernetes_metadata:
              host: ${NODE_NAME}
              matchers:
                - logs_path:
                    logs_path: "/var/log/containers/"
    output.logstash:
      hosts: ["logstash:5044"]
---
apiVersion: apps/v1
kind: DaemonSet              # un pod par nœud automatiquement
metadata:
  name: filebeat
  namespace: homelab
spec:
  template:
    spec:
      serviceAccountName: filebeat
      containers:
        - name: filebeat
          image: docker.elastic.co/beats/filebeat:8.13.0
          env:
            - name: NODE_NAME
              valueFrom:
                fieldRef: { fieldPath: spec.nodeName }
          volumeMounts:
            - { name: varlog,       mountPath: /var/log,                        readOnly: true }
            - { name: containerlog, mountPath: /var/lib/docker/containers, readOnly: true }
      volumes:
        - name: varlog
          hostPath: { path: /var/log }           # logs du nœud hôte
        - name: containerlog
          hostPath: { path: /var/lib/docker/containers }
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: filebeat
rules:
  - apiGroups: [""]
    resources: ["pods", "namespaces", "nodes"]
    verbs: ["get", "list", "watch"]

Ingress Traefik + TLS Let's Encrypt

Toutes les routes HTTP sont déclarées dans un seul Ingress. cert-manager gère automatiquement les certificats TLS via Let's Encrypt pour le domaine public. Les services internes (.home) sont accessibles en local uniquement.

ingress.yaml YAML 
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: homelab-ingress
  namespace: homelab
  annotations:
    kubernetes.io/ingress.class: "traefik"
    traefik.ingress.kubernetes.io/router.entrypoints: websecure
    cert-manager.io/cluster-issuer: letsencrypt-prod
spec:
  tls:
    - hosts: [root.fariadossantos.com]
      secretName: root-tls            # cert auto-renew via cert-manager
  rules:
    - host: kibana.home
      http:
        paths:
          - { path: /, pathType: Prefix, backend: { service: { name: kibana, port: { number: 5601 } } } }
    - host: jellyfin.home
      http:
        paths:
          - { path: /, pathType: Prefix, backend: { service: { name: jellyfin, port: { number: 8096 } } } }
    - host: grafana.home
      http:
        paths:
          - { path: /, pathType: Prefix, backend: { service: { name: grafana, port: { number: 3000 } } } }
    - host: root.fariadossantos.com      # domaine public avec TLS
      http:
        paths:
          - { path: /, pathType: Prefix, backend: { service: { name: root-svc, port: { number: 80 } } } }

Stack technique

Kubernetes Orchestration : single node Ubuntu 22.04
SMB CSI driver Montage NAS FreeNAS (192.168.1.240) en PVC
ELK 8.13 Elasticsearch StatefulSet + Logstash + Kibana
Prometheus + Grafana Métriques node-exporter + dashboards
rclone + CronJob Backup quotidien NAS → Google Drive
Traefik + cert-manager Ingress + TLS auto Let's Encrypt
