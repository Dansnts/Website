Pipeline Distribuée d'Annotation Automatique d'Images Géospatiales

Travail de Bachelor · HEIG-VD 2025-26 · Superviseur : Prof. Bertil Chapuis

En cours

Contexte

L'Institut IICT de la HEIG-VD conduit le projet NearAI, dont l'objectif est de construire une base de données géospatiale d'éléments routiers à partir d'acquisitions mobiles. Les images sont capturées par des véhicules équipés d'un Trimble MX50 ou d'une GoPro Max et produisent des panoramas équirectangulaires allant jusqu'à 8'192 × 4'096 pixels, accompagnés de coordonnées GPS enregistrées dans l'EXIF. Le corpus cible dépasse 300'000 images. Annoter manuellement ce volume représenterait plus de 2'500 heures de travail : l'annotation automatisée est la seule voie viable.

Problème

Deux classes d'objets sont ciblées : les panneaux de signalisation (sign) et les marquages au sol (road_marking). Chaque image à 8'192 × 4'096 pixels dépasse la fenêtre d'entrée de tout modèle de segmentation courant, il faut découper en tuiles avant l'inférence. Traiter 300'000 images en un délai raisonnable impose de distribuer le calcul sur plusieurs GPUs en parallèle.

Pipeline en 5 étapes

  • Lecture : le driver liste les objets du préfixe S3 d'entrée et distribue les clés d'image aux workers Ray.
  • Téléchargement : chaque worker télécharge l'image depuis MinIO et extrait les coordonnées GPS de l'EXIF.
  • Inférence : l'image est découpée en tuiles 512 × 512 px (après downsampling optionnel à 50%), chaque tuile est passée à SAM3 en mode everything.
  • Agrégation : les masques produits sont convertis en polygones, normalisés en pourcentage des dimensions de l'image, filtrés par score de confiance.
  • Écriture : les polygones sont sérialisés dans un fichier Parquet (compression Snappy) et envoyés sur MinIO.

Architecture technique

Cluster GPU iict-rad · K8s 1.33.9 · 9 GPUs (L40S × 3, A40 × 2 actifs)
Calcul distribué Ray 2.54.0 + KubeRay (RayCluster, Actors GPU)
Modèle IA SAM3 ViT-H (Meta, 2024) · 3.3 GB · mode everything
Stockage objet MinIO sur Synology SA3200D, protocole S3
Format de sortie Parquet (Snappy) · 9 colonnes : label, score, polygone, GPS, dimensions
Validation humaine Label Studio · import pré-annotations JSON via API REST
Observabilité Prometheus · Grafana · Loki · Alloy · DCGM (métriques GPU)
Runtime Python 3.12 · PyTorch 2.7.0 · CUDA 12.6 · boto3 · PyArrow

Réalisations

  • Image Docker publiée sur GHCR (ghcr.io/nearai-interreg/ray-sam3:latest) basée sur nvidia/cuda:12.6.3
  • Pipeline Python (sam3_minio_pipeline.py) en mode batch (Parquet) et solo (JSON Label Studio)
  • RayCluster déployé via KubeRay avec nodeAffinity sur nœuds L40S et A40, credentials MinIO et HuggingFace injectés via Secrets
  • Sortie Parquet avec GPS EXIF, polygones normalisés et score de confiance SAM3, queryable directement via DuckDB ou PyArrow
  • Conversion Parquet → JSON Label Studio et intégration pré-annotations via API REST
  • Stack d'observabilité complète : DCGM Exporter, Prometheus, Grafana Alloy, Loki, dashboards Grafana GPU + logs
  • Tests validés sur données réelles : 40 images (dataset Samples) et 2'000 images (dataset HSN)
