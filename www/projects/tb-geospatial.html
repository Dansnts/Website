Pipeline Distribué d'Annotation d'Images Panoramiques par IA

Travail de Bachelor - HEIG-VD 2026

En cours

Contexte

L'annotation automatique d'objets dans de vastes collections d'images panoramiques haute définition est une tâche complexe nécessitant une puissance de calcul importante. Ce projet de Bachelor propose de concevoir et implémenter un pipeline de traitement distribué, exploitant le modèle d'IA SAM3 (Segment Anything Model 3), pour générer des annotations vectorielles de haute qualité sur 300 000 images. Le déploiement s'effectue sur le cluster Kubernetes de l'école en utilisant le framework Ray pour orchestrer les calculs.

Objectif

Développer une plateforme robuste et optimisée capable d'ingérer des images HD (8000x4000 px), d'exécuter à grande échelle un modèle de segmentation avancé (SAM3), et de stocker les annotations géométriques résultantes dans une base de données géospatiale, tout en étudiant les compromis performance/qualité via des techniques de downsampling.

Travail attendu

  • Déployer et configurer un environnement de calcul distribué avec Ray sur Kubernetes
  • Intégrer le modèle SAM3 au sein de tâches Ray et concevoir un pipeline d'annotation par lots
  • Mettre en place un stockage objet S3 pour les images sources, intermédiaires et les masques bruts
  • Implémenter une stratégie de downsampling et analyser son impact sur la qualité et les performances
  • Développer un module de post-traitement pour convertir les masques SAM3 en géométries vectorielles (polygones)
  • Concevoir et peupler un schéma PostgreSQL/PostGIS pour le stockage et l'interrogation spatiale des annotations
  • Évaluer le pipeline en termes de scalabilité, de précision et d'efficacité des ressources

Architecture technique

Orchestration Kubernetes (cluster HEIG-VD)
Calcul distribué Ray Framework
Modèle IA SAM3 (Segment Anything Model 3)
Stockage images Object Storage compatible S3
Base de données PostgreSQL / PostGIS
Données Images HD (8000×4000) → Polygones vectoriels

Livrables

  • Pipeline de traitement distribué complet et fonctionnel (ingestion S3 → écriture PostGIS)
  • Base de données PostGIS avec jeu d'annotations de démonstration géoréférencées
  • Étude comparative documentée sur les stratégies de downsampling (qualité vs. performance)
  • Scripts de déploiement, de monitoring et de validation des résultats
  • Rapport final : architecture, choix techniques, analyse des performances et recommandations
