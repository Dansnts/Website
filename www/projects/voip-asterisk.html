Infrastructure VoIP Multi-sites

Système de téléphonie inter-sites avec Asterisk - Projet CFC ETML

Contexte

Projet réalisé durant mon CFC à l'ETML sous la supervision de M. Karim Bourahla. Chaque groupe d'étudiants était responsable d'un site géographique différent. J'étais en charge du site de Lausanne et devais configurer l'infrastructure pour communiquer avec les autres sites.

Objectifs

  • Déployer un serveur Asterisk pour la téléphonie VoIP
  • Configurer la communication inter-sites via SIP
  • Mettre en place un plan de numérotation cohérent
  • Assurer la qualité de service (QoS) pour les appels

Flux d'appel SIP inter-sites

Sequence de signalisation SIP lors d'un appel entre le poste 1001 du site Lausanne et le poste 2001 du site Geneva. Le media audio (RTP) s'etablit en direct entre les endpoints une fois la negociation SDP terminee.

sequenceDiagram participant Phone1 as Telephone 1001\nLausanne participant AstLSN as Asterisk Lausanne\n10.0.1.10 participant AstGVA as Asterisk Geneva\n10.0.2.10 participant Phone2 as Telephone 2001\nGeneva Phone1->>AstLSN: INVITE sip:2001@lausanne (SDP offer) AstLSN->>AstLSN: Lookup dialplan: exten 2001 => trunk-gva AstLSN->>AstGVA: INVITE sip:2001@10.0.2.10 (SDP offer) AstGVA->>AstGVA: Lookup dialplan: exten 2001 => local AstGVA->>Phone2: INVITE sip:2001@phone2 (SDP offer) Phone2-->>AstGVA: 100 Trying Phone2-->>AstGVA: 180 Ringing AstGVA-->>AstLSN: 180 Ringing AstLSN-->>Phone1: 180 Ringing Phone2-->>AstGVA: 200 OK (SDP answer) AstGVA-->>AstLSN: 200 OK (SDP answer) AstLSN-->>Phone1: 200 OK (SDP answer) Phone1->>AstLSN: ACK AstLSN->>AstGVA: ACK AstGVA->>Phone2: ACK Note over Phone1,Phone2: Media RTP audio direct (G.711 ulaw) Phone1->>AstLSN: BYE AstLSN->>AstGVA: BYE AstGVA->>Phone2: BYE Phone2-->>AstGVA: 200 OK AstGVA-->>AstLSN: 200 OK AstLSN-->>Phone1: 200 OK

Plan de numerotation Asterisk

Extrait du fichier extensions.conf du site Lausanne. Les extensions locales (1XXX) sont gerees en interne ; les appels vers les autres sites utilisent les trunks SIP definis dans sip.conf.

extensions.conf Asterisk Dialplan 
[general]
static=yes
writeprotect=no
autofallthrough=yes

; ─── Extensions locales site Lausanne (1XXX) ───────────────────────────────
[lausanne-local]
; Postes SIP locaux
exten => 1001,1,Dial(SIP/1001,20,tr)
exten => 1001,2,VoiceMail(1001@default,u)
exten => 1001,n,Hangup()

exten => 1002,1,Dial(SIP/1002,20,tr)
exten => 1002,2,VoiceMail(1002@default,u)
exten => 1002,n,Hangup()

; Messagerie vocale
exten => *97,1,VoiceMailMain(${CALLERID(num)}@default)
exten => *97,n,Hangup()

; ─── Trunk SIP vers Geneva (2XXX) ──────────────────────────────────────────
[lausanne-outbound]
exten => _2XXX,1,NoOp(Appel vers Geneva: ${EXTEN})
exten => _2XXX,n,Dial(SIP/trunk-geneva/${EXTEN},30,tr)
exten => _2XXX,n,Congestion()
exten => _2XXX,n,Hangup()

; Trunk SIP vers Bern (3XXX)
exten => _3XXX,1,NoOp(Appel vers Bern: ${EXTEN})
exten => _3XXX,n,Dial(SIP/trunk-bern/${EXTEN},30,tr)
exten => _3XXX,n,Congestion()
exten => _3XXX,n,Hangup()

; ─── Trunk entrant depuis autres sites ─────────────────────────────────────
[from-trunk]
; Appels entrants : router vers les extensions locales
exten => _1XXX,1,Goto(lausanne-local,${EXTEN},1)
exten => _1XXX,n,Hangup()
sip.conf (extrait trunk) Asterisk SIP 
[general]
context=from-trunk
bindport=5060
bindaddr=0.0.0.0
transport=udp

; ─── Trunk SIP vers Geneva ──────────────────────────────────────────────────
[trunk-geneva]
type=friend
host=10.0.2.10
port=5060
username=trunk-lsn
secret=s3cret_trunk
context=from-trunk
disallow=all
allow=ulaw
allow=alaw
qualify=yes
insecure=port,invite
dtmfmode=rfc2833

; ─── Poste local 1001 ───────────────────────────────────────────────────────
[1001]
type=friend
host=dynamic
context=lausanne-local
disallow=all
allow=ulaw
callerid="Alice Dupont" <1001>
mailbox=1001@default
nat=yes
