Network Automation

Scripts d'automatisation pour équipements réseau

Vue d'ensemble

Collection de scripts et playbooks pour automatiser la configuration et la gestion d'équipements réseau multi-vendeurs. Réduction du temps de déploiement et minimisation des erreurs humaines.

Fonctionnalités

  • Backup automatique des configurations
  • Déploiement de VLANs en masse
  • Audit de conformité des configurations
  • Génération de rapports d'inventaire
  • Templates Jinja2 pour configurations standardisées

Pipeline d'automatisation reseau

Workflow Ansible pour la gestion multi-vendeurs : inventaire dynamique, collecte de l'etat courant, application des configurations depuis templates Jinja2, verification de conformite et rapport final.

Playbook Ansible : Deploiement VLAN multi-vendeurs

Exemple de playbook deploiant une liste de VLANs sur l'ensemble du parc. Les templates Jinja2 generent la syntaxe specifique a chaque OS cible.

deploy_vlans.yml YAML 
---
- name: Deploy VLANs - Cisco IOS-XE
  hosts: cisco
  gather_facts: false
  vars:
    vlans:
      - { id: 100, name: "PRODUCTION" }
      - { id: 200, name: "MANAGEMENT" }
      - { id: 300, name: "DMZ" }

  tasks:
    - name: Backup running-config
      cisco.ios.ios_config:
        backup: true
      register: backup_output

    - name: Configure VLANs
      cisco.ios.ios_vlans:
        config:
          - vlan_id: "{{ item.id }}"
            name: "{{ item.name }}"
            state: active
        state: merged
      loop: "{{ vlans }}"

    - name: Verify VLANs present
      cisco.ios.ios_command:
        commands: show vlan brief
      register: vlan_output

    - name: Assert VLAN visible
      ansible.builtin.assert:
        that: "'{{ item.name }}' in vlan_output.stdout[0]"
        fail_msg: "VLAN {{ item.name }} absent de la table"
      loop: "{{ vlans }}"

---
- name: Deploy VLANs - Mikrotik RouterOS
  hosts: mikrotik
  gather_facts: false
  vars:
    vlans:
      - { id: 100, name: "PRODUCTION", interface: "bridge" }
      - { id: 200, name: "MANAGEMENT", interface: "bridge" }

  tasks:
    - name: Create VLAN bridge ports
      community.routeros.api:
        path: interface bridge vlan
        add:
          bridge: bridge
          vlan-ids: "{{ item.id }}"
          tagged: "{{ item.interface }}"
          comment: "{{ item.name }}"
      loop: "{{ vlans }}"

---
- name: Audit compliance - tous vendeurs
  hosts: all
  gather_facts: false
  tasks:
    - name: Collect device info
      ansible.builtin.include_role:
        name: network_audit
      vars:
        output_dir: "./reports/{{ inventory_hostname }}"
